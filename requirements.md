# 政策长图生成工具 - 需求文档（V1.0）

## 1. 项目背景

将政策PDF文档转化为蓝色政务风格长图，供政务传播使用。

### 参考输入
- **PDF文件**：`/Users/sjs/Downloads/05B20260430C_l.pdf`（关于"三新"商业保险的政策文章）
- **风格参考**：`/Users/sjs/Downloads/640.jpeg`（蓝色政务长图模板）

### 参考风格要素
- 配色：蓝白为主，政务严肃风格

## 2. 功能范围（V1）

### 2.1 文件上传
- 支持 **PDF** 和 **DOCX** 格式上传
- 单文件大小限制：20MB
- 基础错误提示（文件类型错误、文件过大、上传失败）

### 2.2 文本抽取
- 从 PDF/DOCX 中自动提取标题、正文、关键段落
- 处理纯文本内容（暂不支持图片内文字识别）

### 2.3 结构化摘要
- 自动识别并生成：
  - 文章标题
  - 核心摘要（200字以内）
  - 关键要点（3-6条，每条不超过50字）
  - 来源/出处信息

### 2.4 蓝色政务风长图生成
- **固定模板**，不开放模板编辑
- 长图结构：
  ```
  ┌──────────────────────────┐
  │         顶部装饰条        │  蓝色渐变
  ├──────────────────────────┤
  │         标题区             │  文章标题，居中
  ├──────────────────────────┤
  │        正文摘要           │  核心内容，3-5行
  ├──────────────────────────┤
  │      分条政策卡片         │  关键要点卡片列表
  │  ┌────────────────────┐  │
  │  │ 要点1               │  │
  │  ├────────────────────┤  │
  │  │ 要点2               │  │
  │  ├────────────────────┤  │
  │  │ 要点3               │  │
  │  └────────────────────┘  │
  ├──────────────────────────┤
  │        底部说明           │  来源、日期等
  │        底部装饰条         │  蓝色
  └──────────────────────────┘
  ```
- 宽度：1080px（适合社交传播）
- 高度：根据内容自动延伸（约2000-4000px）

### 2.5 浏览器预览
- 在浏览器中直接预览生成的长图（PNG格式渲染）

### 2.6 PNG 下载
- 一键下载生成的长图PNG文件

## 3. 技术方案

### 3.1 技术栈
- **后端**：Python + FastAPI
- **PDF解析**：`PyMuPDF`（或 `pdfplumber`）
- **DOCX解析**：`python-docx`
- **长图生成**：`Pillow`
- **前端**：HTML + CSS（简洁单页，无框架依赖）
- **部署**：Docker 或直接部署

### 3.2 系统架构
```
用户上传 PDF/DOCX
       ↓
后端文本抽取 → 结构化摘要生成（规则 + 正则）
       ↓
HTML模板渲染（标题、摘要、要点、底部）
       ↓
Pillow 渲染为 PNG
       ↓
返回预览链接 / 下载
```

### 3.3 核心流程
1. 用户上传文件 → 服务端保存临时文件
2. 文本抽取 → 提取标题、段落
3. 摘要生成 → 按规则提取核心句、关键点
4. 长图渲染 → 使用 Pillow 将模板+数据渲染为 PNG
5. 返回预览/下载

## 4. 非功能约束

### 4.1 无账号体系
- 不需要登录注册
- 访客直接使用

### 4.2 无复杂模板编辑
- 模板固定，不开放样式修改
- 仅通过数据填充生成

### 4.3 错误处理
- 文件类型不支持 → 提示"仅支持 PDF、DOCX 格式"
- 文件过大 → 提示"文件大小不超过 20MB"
- 解析失败 → 提示"文件解析失败，请检查文件内容"
- 空内容 → 提示"未检测到有效内容"
